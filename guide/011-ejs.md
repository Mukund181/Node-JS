# EJS (Embedded JavaScript Templates)

EJS is a simple templating language that lets you generate HTML markup with plain JavaScript, serving as the View layer in MVC (Model-View-Controller) Express applications.

---

## 1. Prerequisites & Conceptual Basics

### SSR (Server-Side Rendering) vs CSR (Client-Side Rendering)
- **CSR (e.g. React/Vue)**: The server returns a barebones HTML file alongside a massive Javascript bundle. The client browser executes this Javascript to build the UI page in real time.
- **SSR (e.g. EJS, Pug)**: The server fetches data, parses a template, computes all conditional variables (loops, if/else statements), compiles a complete HTML string, and sends it back. The client browser receives pre-rendered markup immediately, which improves initial load times and search engine indexing (SEO).

---

## 2. Theory & Deep Dive

### View Pathing Architecture
By default, Express looks for template files inside a directory named `/views` in the project root. You can customize this by setting the views path configuration:
```javascript
app.set("views", path.join(__dirname, "templates"));
```

### Layout Partials
EJS allows you to extract shared UI layouts (like a navigation bar, header, or footer) into standalone `.ejs` files and import them across multiple views, helping you adhere to DRY (Don't Repeat Yourself) design principles.

## 3. Code Implementation

To run EJS, first install the dependency: `npm install ejs`

Create a folder structure:
```text
project/
├── server.js
└── views/
    ├── partials/
    │   └── header.ejs
    └── index.ejs
```

### File: `views/partials/header.ejs`
```html
<header style="background: navy; color: white; padding: 10px;">
  <h2><%= siteName %></h2>
</header>
```

### File: `views/index.ejs`
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <title>EJS Home</title>
</head>
<body>
  <!-- Include partial layout -->
  <%- include('partials/header') %>

  <h1>Welcome, <%= user.username %>!</h1>

  <% if (user.role === 'admin') { %>
    <p style="color: red;"><strong>Admin Dashboard Access Enabled</strong></p>
  <% } else { %>
    <p>User Dashboard Access Enabled</p>
  <% } %>

  <h3>Active Task List</h3>
  <ul>
    <% tasks.forEach(task => { %>
      <li><%= task %></li>
    <% }) %>
  </ul>
</body>
</html>
```

### File: `server.js` (Express Server)
```javascript
const express = require("express");
const path = require("path");
const app = express();

const PORT = 3000;

// Set EJS as the application template view engine
app.set("view engine", "ejs");

// Set directory for views (defaults to "views")
app.set("views", path.join(__dirname, "views"));

app.get("/", (req, res) => {
  const data = {
    siteName: "ConnectSpace Tech",
    user: {
      username: "Mukund181",
      role: "admin"
    },
    tasks: ["Complete Node.js Guide", "Deploy to Production", "Implement OAuth"]
  };

  // Render index.ejs template and pass the data context object
  res.render("index", data);
});

app.listen(PORT, () => {
  console.log(`EJS Server running on http://localhost:${PORT}`);
});
```

## 4. Self-Contained Mini-Project: Product Inventory Manager

We will build a simple dynamic dashboard to manage and filter a list of store products.

### Project Setup

```text
└── views/
    ├── partials/
    │   └── nav.ejs
```

### File: `views/partials/nav.ejs`

```html
<nav style="background:#333; padding:15px; color:#fff; display:flex; justify-content:space-between;">

  <span>User: Administrator</span>
```

### File: `views/inventory.ejs`

```html
                <span class="warning">OUT OF STOCK</span>
              <% } else if (p.stock < 5) { %>
                <span style="color:orange;">LOW STOCK</span>
              <% } else { %>
                <span style="color:green;">IN STOCK</span>
              <% } %>
            </td>
          </tr>
        <% }) %>
      </tbody>
    </table>

```

### File: `server.js`
```javascript
const express = require("express");
const path = require("path");
const app = express();

app.set("view engine", "ejs");
app.set("views", path.join(__dirname, "views"));

const mockDb = [
  { name: "Wireless Mouse", stock: 12 },
  { name: "HDMI Cable", stock: 3 },
  { name: "USB Keyboard", stock: 0 },
  { name: "Mechanical Keyboard", stock: 15 }
];

app.get("/inventory", (req, res) => {
  const searchWord = req.query.search || "";
  
  // Filter inventory
  const filteredProducts = mockDb.filter(p => 
    p.name.toLowerCase().includes(searchWord.toLowerCase())
  );

  res.render("inventory", {
    products: filteredProducts,
    searchWord: searchWord
  });
});

app.listen(3000, () => console.log("Inventory dashboard running on http://localhost:3000/inventory"));
```

## 5. Advanced Production Practices & Security

### Cross-Site Scripting (XSS) Vulnerabilities
If you output variables using the raw unescaped tag `<%- %>`, any HTML or `<script>` tags stored in that variable will be rendered directly by the browser. If a user inputs `<script>stealCookies()</script>` into a form and you render it using `<%- %>`, visitors will execute the script, exposing them to XSS attacks.
- **Solution**: Always use `<%= %>` for outputting user-generated text. It automatically escapes HTML entities, converting `<` to `&lt;`. Only use `<%- %>` for rendering trusted server-generated HTML (like templates).

### View Cache in Production
Compiling templates repeatedly during high-traffic requests causes CPU overhead. In production, turn on view caching to store compiled templates in memory:
```javascript
app.set("view cache", true);
```
*(This is enabled automatically if `process.env.NODE_ENV` is set to `"production"`).*

---

## 6. Key Takeaways
1. **EJS Tag reference**:
   - `<% %>` Control flow tag (for loops, if conditions, no output).
   - `<%= %>` Escaped output tag (prints values safely, converting HTML characters).
   - `<%- %>` Unescaped output tag (renders raw HTML; use carefully to prevent XSS).
   - `<%- include('filename') %>` Includes template partial layouts.
2. EJS is rendered entirely on the server; the client browser receives raw compiled HTML and never sees the backend template script tags.
